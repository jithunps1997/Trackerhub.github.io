Reports

User Report
User reports contains information on system activities over a given period of time. A content manager can only run the user reports.
Admin Report
Administrative reports provide overview information about your customers, their systems and the status of agents and probes. The reports available depend upon whether you have accessed the page from the Service Organization (SO) or Customer or Site (CS) level.
Daily Login Report
The Daily IT Login Report should include the following details for each IT personnel's login: Date and Time: The timestamp of the login activity. User: The username or ID of the IT personnel. IP Address: The IP address from which the login originated. Device: The device or workstation used for the login. Location: The general geographical location (city or region) of the login. 2. Reporting Frequency Generate the Daily IT Login Report at the end of each day. This ensures timely analysis of login activities and identification of any suspicious or unauthorized logins. 3. Data Sources Collect data for the report from: Authentication Logs: Use authentication logs from the central authentication system, which includes details of successful and failed login attempts. 4. Data Retention Retain daily login records for a defined retention period. This helps with auditing and historical analysis. Ensure compliance with data retention policies and relevant regulations. 5. Security and Access Limit access to the Daily IT Login Report to authorized personnel only. Use appropriate security controls to protect the report from unauthorized access or tampering. 6. Analysis and Review Regularly review the Daily IT Login Report to identify any unusual patterns or suspicious activities. Investigate any unauthorized or unexpected logins promptly. 7. Reporting In case of suspicious activities or security incidents, report to the appropriate security or incident response team. Maintain a process to escalate and address potential security breaches. 8. Automation (Optional) Consider automating the report generation process using scripting or tools. Automation ensures consistent and timely reporting. 9. Documentation Document the procedures for generating and reviewing the Daily IT Login Report. Include details on how to interpret the report and take appropriate actions. 10. Continuous Improvement Regularly review and update the Daily IT Login Report procedures to adapt to changing security needs. Remember that the Daily IT Login Report contributes to maintaining a secure IT environment by identifying and addressing potential security risks related to login activities.
